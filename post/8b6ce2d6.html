hexo安装和github部署配置

发表于 2019-06-11

下载安装git

下载地址：https://www.git-scm.com/download/win

安装nodejs 和npm

  • linux下执行命令：

  • windows下需要下载相应的window版本：

　　https://nodejs.org/en/

安装hexo

npm install hexo-cli -g

生成项目目录blog

hexo init blog
cd blog
npm install hexo-deployer-git -save

###在blog文件夹下生成 package.json文件

在blog文件夹下执行

安装package.json指定的nodejs模块

在blog文件夹下执行

git clone https://github.com/theme-next/hexo-theme-next themes/next

启用主题

打开站点配置文件_config.yml， 找到 theme 字段，并将其值更改为 next

_config.yml文件配置

启动本地服务器

hexo server -p 5000

打开主题配置文件_config.yml,设置Scheme

  • Muse 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白

  • Mist Muse 的紧凑版本，整洁有序的单栏外观

  • Pisces 双栏 Scheme，小家碧玉似的清新

设置语言

编辑 站点配置文件， 将 language 设置成你所需要的语言。

language: zh-Hans

设置头像

　编辑主题配置文件

##avatar: https://i.imgur.com/BjSGuy2.png
##文件本地位置为:source/images/logo.jpg
avatar: /images/logo.jpg

sitemap、rss 和部署

　　切换到blog根目录下，输入：

npm install hexo-generator-feed
npm install hexo-generator-sitemap

　　之后重启博客，访问/atom.xml和/sitemap.xml，会发现已经生成了。可以把sitemap提交到搜索引擎的站长平台来增加收录。

GitHub中Hexo next主题下搭建的博客中增加PDF插件

  • 在博客bash中执行下列命令安装

  • 外部文章网页链接

    {% pdf http://7xov2f.com1.z0.glb.clouddn.com/bash_freshman.pdf %}

  • 本地在博客source_posts\，文件夹下创建一个文章如test.md,同时需要再创建一个同名的test
    文件夹，放我们要上传的PDF文章test.pdf,并在test.md中加入：

    {% pdf  test.pdf %}

创建tag页面

  • 新建一个页面，命名为 tags。命令如下：

    hexo new page tags

  • 编辑刚新建的页面，将页面的类型设置为 tags ，主题将自动为这个页面显示标签云。页面内容如
    下：

    title: Tagcloud
date: 1/30/2018 5:33:34 PM
type: tags

hexo 相册功能

无论哪种方法都有一个共同的前提：修改_config.yml配置文件

post_asset_folder：true

新建一个博客

hexo new 我的相册

　　在_posts下会生成 我的相册 文件夹,将图片放入 我的相册 文件夹
修改文件 我的相册.md,添加如下内容

{% asset_img 图片.jpg 图片的说明 %}
